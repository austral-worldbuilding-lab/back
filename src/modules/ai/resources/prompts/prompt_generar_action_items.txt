OBJETIVO:
  Generar action items específicos, concretos y accionables para implementar la solución propuesta. El propósito es descomponer la solución en pasos prácticos y ordenados que permitan su ejecución efectiva, identificando acciones claras con responsabilidades y tiempos estimados.

CONFIGURACIÓN DEL MUNDO:
  • Nombre del Mundo: ${projectName}
  • Descripción del Mundo: ${projectDescription}

CONTEXTO DE PROVOCACIONES:
  • Para llegar a este mundo, se analizaron las provocaciones: ${provocations-timeline}

SOLUCIÓN A IMPLEMENTAR:
  ${solution}

CONFIGURACIÓN DE LA GENERACIÓN:
  • Cantidad mínima de action items: ${minActionItems}
  • Cantidad máxima de action items: ${maxActionItems}

REGLAS PARA LOS ACTION ITEMS:
  • El campo `order` debe ser un número secuencial (1, 2, 3...) que indique el orden lógico de ejecución de los action items.
  • El campo `title` debe ser un título corto y descriptivo de la acción a realizar (máximo 8-10 palabras).
  • El campo `description` debe explicar en detalle qué implica realizar esta acción, qué se debe hacer específicamente, qué recursos se necesitan, y qué resultado se espera obtener.
  • El campo `duration` es opcional y debe contener una estimación de tiempo realista para completar esta acción (ej: "1 mes", "2 semanas", "3 meses", "15 días").
  • Los action items deben ser ESPECÍFICOS y CONCRETOS, no genéricos ni abstractos.
  • Cada action item debe ser ACCIONABLE: debe quedar claro qué hacer, quién debería hacerlo (rol/área), y cómo se vería el resultado.
  • Los action items deben estar ordenados lógicamente según la secuencia de implementación: primero las acciones de preparación, luego las de ejecución, y finalmente las de consolidación.
  • Evitar acciones demasiado genéricas como "hacer seguimiento" o "coordinar equipo". En su lugar, ser específico sobre QUÉ se debe coordinar o hacer seguimiento.
  • Si la solución menciona múltiples componentes o fases, generar action items que cubran todos esos aspectos de forma equilibrada.
  • Los action items deben considerar el contexto del mundo y ser viables dentro de sus particularidades.

FORMATO DE RESPUESTA:
  Debes responder con un array JSON de objetos. Cada objeto debe tener esta estructura:
  {
    "order": <número>,
    "title": "<título corto>",
    "description": "<descripción detallada>",
    "duration": "<duración estimada>" // OPCIONAL
  }
